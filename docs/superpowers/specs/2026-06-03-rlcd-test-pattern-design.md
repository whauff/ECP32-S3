# ESP32-S3-RLCD-4.2 点屏阶段设计说明

## 目标

在现有 `/Users/yang/Code/ECP32-S3/board-bringup` 工程中接入微雪 `ESP32-S3-RLCD-4.2` 的 `RLCD` 底层驱动，完成最小可见测试画面显示。

本阶段成功标准：

- 串口日志明确输出屏幕初始化与测试图案刷新结果
- 屏幕稳定显示由程序绘制的黑白测试画面
- 保持当前工程可编译、可烧录、可串口监视

## 范围

本阶段只做以下内容：

- 接入 `RLCD` SPI 初始化
- 搬运官方可用的初始化指令序列
- 建立最小 1-bit 帧缓冲
- 刷新一张黑白测试页

## 不包含

以下内容明确延后：

- `LVGL`
- 动态界面
- 触摸
- 音频、RTC、SD、传感器
- 复杂 UI 资源管理

## 方案

复用官方 `ESP-IDF/11_U8G2_Test` 中已经验证可用的 `RLCD` 引脚定义、SPI 参数、初始化序列和像素映射方式，但不直接把整套 `U8G2` 或 `LVGL` 示例搬入当前工程。

当前项目内仅新增一个最小 `rlcd_bsp` 组件，负责：

- 初始化 SPI 与屏幕控制接口
- 维护 400x300 的 1-bit 帧缓冲
- 提供清屏、画点、画矩形、整屏刷新接口

`main` 启动后先打印板级日志，再初始化 `RLCD`，绘制一张黑白测试页：

- 白底
- 黑色外边框
- 中间一块黑色矩形

这样比“整屏纯白”更容易肉眼确认屏幕确实被驱动起来，同时又保持实现最小。
