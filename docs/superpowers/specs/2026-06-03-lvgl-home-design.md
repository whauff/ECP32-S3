# LVGL 最小首页设计说明

## 目标

在现有 `board-bringup` 工程和已验证通过的 `RLCD` 驱动基础上，接入最小可用的 `LVGL` 显示链路，并输出一个稳定的首页界面，替代当前的测试图形页。

本阶段只验证：

- `LVGL` 初始化成功
- `LVGL` 刷新回调能驱动 `RLCD`
- 开机后能显示一个稳定、可读的黑白首页

本阶段不包含：

- 时间同步
- 电池真实采集
- 传感器数据接入
- 复杂动画
- 多页面导航

## 方案

采用“保留现有 `RLCD` 底层 + 新增轻量 `LVGL Port` + 最小首页 UI”的方式推进。

- `rlcd_bsp` 继续负责屏幕初始化、1-bit 帧缓冲维护和最终上屏
- `lvgl_port` 负责 `LVGL` 的 `tick`、任务循环、显示驱动注册和局部刷新
- 首页 UI 只包含标题、时间占位、电量占位和状态文案

为了避免大显存带来的内存压力，本阶段不使用整屏双缓冲，而是使用较小的 `RGB565` 局部缓冲区。`LVGL` 每次刷新时，只把变更区域转换为黑白像素并写入 `rlcd_bsp` 的持久帧缓冲，等一轮刷新结束后再统一提交到屏幕。

## 界面内容

首页采用黑白高对比布局，适配反射屏：

- 顶部标题：`ECP32-S3`
- 中间主信息：`12:30`
- 辅助信息：`LVGL Ready`
- 底部状态：`屏幕链路正常`
- 右上角状态角标：`BAT 100%`

## 成功标准

- 串口日志出现 `LVGL` 初始化完成
- 开机后屏幕显示首页，而不是测试矩形页
- 画面稳定，无明显花屏或持续闪烁
- 编译、烧录、串口验证可重复执行
